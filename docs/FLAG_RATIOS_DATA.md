# Flag Ratios Data

Official aspect ratios (width : height) for every country's flag.

> Format: `width:height` — e.g., `2:3` means width is 2, height is 3 (portrait-leaning).
> For display in the app, the FlagView renders PDF assets which already encode the correct ratio.
> This data is useful for: (a) quiz questions about flag shapes, (b) ensuring correct aspect ratio in flag display containers, (c) the "Flag Ratios" quiz type.

---

## Special Cases (Non-Standard Ratios)

### Nepal 🇳🇵
- **Ratio**: Non-rectangular — double pennant (two stacked triangular pennants)
- **Dimensions**: Height ≈ 12 units, width ≈ 9.5 units at widest
- **Only non-rectangular national flag in the world**
- **Construction**: Codified in Schedule 1 of the 1990 Constitution with precise geometric instructions

### Switzerland 🇨🇭
- **Ratio**: 1:1 (square)
- **Note**: One of only two square national flags (the other being Vatican City)

### Vatican City 🇻🇦
- **Ratio**: 1:1 (square)

---

## Very Wide Flags

### Qatar 🇶🇦
- **Ratio**: 11:28 (very wide — almost 1:2.5)
- **Note**: Widest national flag proportionally; the maroon/white serrated division is distinctive

### Togo 🇹🇬
- **Ratio**: Golden ratio (1:φ ≈ 1:1.618)
- **Note**: Only flag designed using the golden ratio

### Nepal — see above (special shape)

## Implementation Notes for FlagView

The `FlagView(countryCode:height:)` renders PDF assets that inherently preserve aspect ratio. The container calling it should either:

1. Use `FlagView(countryCode: code, height: 40)` and let width auto-size (current approach — correct)
2. For fixed-size containers (e.g., 3x2 grid), crop or add padding for non-standard ratios
3. **Nepal special case**: The PDF asset for Nepal should be rendered without clipping — it will appear inside a bounding box with empty corners

For the "Flag Quiz" feature that asks about flag ratios, query this data from a static dictionary keyed by country code.
